# Step 7：文明数据补充

## 目标

在 step6 的 override 架构就绪后，按用户查询频率和文明复杂度，渐进式补充独特文明的科技映射。

## 文明优先级

| 优先级 | 文明 | 独特程度 | 说明 |
|:------:|:-----|:--------:|:-----|
| P1 | 龙之骑士团 OD | ⭐⭐⭐⭐⭐ | 全特色变体，最需要 |
| P2 | 朱熹遗产 ZX | ⭐⭐⭐⭐ | 朝代切换影响科技池 |
| P3 | 金朝 JN / 金账汗国 GH | ⭐⭐⭐ | 部分特色变体 |
| P4 | 战国大名 SG / 圣殿骑士团 KT | ⭐⭐⭐ | 同上 |
| P5 | 兰开斯特 HL / 马其顿 MD / 图格鲁克 TD | ⭐⭐ | 少数特色 |
| P6 | 其他 Sultans Ascend 文明 | ⭐⭐ | 同上 |

## 每个文明文件的维护流程

```
1. 用户首次查询该文明独特单位
2. step3 自动规则尝试匹配 → 可能有漏/误
3. 发现映射问题 → 编辑 data/tech_data/civ_overrides/<civ>.json
4. 更新 metadata.json 的 updated_at
5. 后续查询直接命中 override
```

## QA 检查清单

每个文明补充后验证：

- [ ] 所有特色单位至少一个时代变体被覆盖
- [ ] 核心科技（铁匠铺、大学、城堡科技）显示正确
- [ ] 文明特色科技（unique=true）正确匹配
- [ ] 跨时代科技在各时代面板中正确显示
- [ ] 排除规则生效（如手炮手不吃铁匠铺远程攻）
- [ ] 图片渲染无排版问题
